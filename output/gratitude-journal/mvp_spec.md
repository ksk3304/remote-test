# 感謝日記アプリ MVP仕様書

## 1. ニッチ定義

| 項目 | 内容 |
|------|------|
| **誰が** | ストレスを抱える社会人（20-40代） |
| **いつ** | 就寝前の3-5分 |
| **何のために** | 日々の感謝を記録し、ポジティブ思考を習慣化 |

## 2. 解決する不満（競合レビューから抽出）

| 競合の問題 | 本アプリの解決策 |
|------------|------------------|
| クラッシュ・バグが多い | シンプル設計で安定性重視 |
| 広告が不快・多い | 広告なし（サブスク課金） |
| 文字数制限がある | 制限なし |
| 過去データが見れない | カレンダーで全期間閲覧可能 |
| データが消える | ローカル保存 + エクスポート機能 |
| SNSが荒れる | SNS機能なし、完全プライベート |

## 3. コア機能（1つだけ）

**「毎日3つの感謝を記録する」**

- 1日1エントリ
- 3つの感謝項目を入力
- 文字数制限なし
- 写真添付なし（シンプル優先）

## 4. Paywall設計

```
[無料版]
├── 3日間のフルお試し
└── 4日目以降：直近7日分のみ閲覧可

[有料版]
├── 全期間のエントリ閲覧
├── カレンダー表示
├── データエクスポート（CSV/JSON）
└── 将来のプレミアム機能
```

**課金タイミング**:
- 7日目以降の過去エントリをタップした時
- 「すべての記録を見る」ボタンをタップした時

## 5. 最小画面構成

### 画面一覧（5画面）

```
[1] ホーム（今日の感謝入力）
    └── 3つの入力フィールド
    └── 保存ボタン
    └── 連続記録日数表示

[2] カレンダー
    └── 月カレンダー表示
    └── 記録した日にマーク
    └── 日付タップで詳細表示

[3] エントリ詳細
    └── 選択した日の3つの感謝を表示
    └── 編集ボタン

[4] 設定
    └── リマインダー設定（時刻）
    └── テーマ（ライト/ダーク）
    └── サブスク管理
    └── データエクスポート
    └── プライバシーポリシー

[5] Paywall
    └── サブスク説明
    └── 価格表示（月額/年額）
    └── 購入ボタン
    └── 復元ボタン
```

## 6. 技術スタック案

### 推奨: SwiftUI（iOSネイティブ）

| 技術 | 選定理由 |
|------|----------|
| SwiftUI | Apple純正、安定、審査通りやすい |
| Core Data | ローカルDB、iCloud同期可能 |
| StoreKit 2 | サブスク課金 |
| UserNotifications | リマインダー |

### 代替: Flutter

| 技術 | 選定理由 |
|------|----------|
| Flutter | クロスプラットフォーム |
| sqflite | ローカルDB |
| in_app_purchase | 課金 |
| flutter_local_notifications | リマインダー |

## 7. 受け入れ基準（MVP完成条件）

### 必須機能

- [ ] 今日の感謝を3つ入力して保存できる
- [ ] カレンダーで過去の記録を確認できる
- [ ] 連続記録日数（ストリーク）が表示される
- [ ] リマインダー通知が設定できる
- [ ] ライト/ダークテーマ切替ができる
- [ ] サブスク購入・復元ができる
- [ ] Paywallが正しく動作する（無料版は7日制限）
- [ ] データをエクスポートできる（CSV）
- [ ] クラッシュしない（安定動作）

### 品質基準

- [ ] 起動から入力完了まで10秒以内
- [ ] 1日の操作時間3分以内を想定したUI
- [ ] オフラインで完全動作
- [ ] App Store審査を通過できる状態
